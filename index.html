The Open Source Definition

Open source doesn't just mean access to the source code.

  • Free Redistribution
  • Source Code
  • Derived Works
  • Integrity of The Author's Source Code
  • No Discrimination Against Persons or Groups
  • No Discrimination Against Fields of Endeavor
  • Distribution of License
  • License Must Not Be Specific to a Product
  • License Must Not Restrict Other Software
  • License Must Be Technology-Neutral
The four most critical pieces of infrastructure that make the Internet work -- Bind, Perl, sendmail and Apache -- every one of these is open source, every one of these is super reliable. The Internet would not function if they weren't super reliable, and they're super reliable precisely because throughout their entire history people have been constantly banging on the code, looking at the source, seeing what breaks and fixing it.
- Eric Raymond
Open Source != Free Software
Free Software Foundation

Richard Stallman aka RMS, The GNU Project

  • 1985 - Free Software Foundation (FSF)
  • Free software means that the software's users have freedom.
“Free software” means software that respects users' freedom and community. Thus, “free software” is a matter of liberty, not price. To understand the concept, you should think of “free” as in “free speech,” not as in “free beer.”

Four essential freedoms:

  • (0) to run the program,
  • (1) to study and change the program in source code form,
  • (2) to redistribute exact copies, and
  • (3) to distribute modified versions.

So what's the difference?

Open Source for Juniors

In open-source projects you can:

  • improve your programming skills
  • gain practical experience on real projects
  • improve your soft skills
  • master Git workflow
How to start
  • Good First Issue
  • Choose a project
  • Learn about the project, explore issues
  • Fork the project and clone it on your computer.
  • Make changes, create good commit descriptions.
  • Synchronize changes with your fork (origin remote).
  • Create a Pull Request on GitHub.
  • Respond to all comments received during the code review.
